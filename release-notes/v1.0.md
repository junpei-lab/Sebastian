# Sebastian 1.0 リリースノート

## ハイライト
- React + Vite フロントエンドと Tauri/Rust バックエンドを組み合わせたクロスプラットフォームのデスクトップアラーム。
- 時刻、タイトル、URL、リードタイム、曜日指定で柔軟に登録できるアラーム一覧、つねに最前面で表示されるアラームダイアログ、ベル音再生。
- アプリを閉じてもトレイに残り、ダブルクリック/メニューから復帰。JSON インポートではバリデーションと既存アラーム置換オプションを提供。
- `alarms.json` はプラットフォーム固有のアプリデータフォルダに保存され、繰り返しアラームは自動で次回発火時刻へ移行。責任ある `parking_lot::Mutex` 制御と `tokio` ループで 1 秒間隔の監視。
- アラーム発火時にウィンドウが最小化されていても `unminimize` → `show` → `set_focus` されて見逃しを防止。

## マニュアルテスト
1. `npm run tauri:dev` でデスクトップ版を起動し、単発/繰り返しアラームを追加・削除できること。
2. URL 付きアラームで「リンクを開く」ボタンが標準ブラウザを起動すること。
3. リードタイムと曜日の設定が UI・保存に反映され、未選択で送信不可になっていること。
4. 設定した時間にアラームが鳴り、ウィンドウが最小化中でも復帰して通知されること。
5. JSON インポート機能で配列・`{ "alarms": [...] }` 形式を読み込めること（置換オプション含む）。
6. タスクトレイの「終了」で終了、トレイアイコンのダブルクリック/表示メニューで復帰すること。
7. `src-tauri/src/main.rs` のアラームループで `alarm_triggered` イベントが送られること。

## 開発者向けメモ
- `src-tauri/src/alarm_store.rs` を更新する際は Rust テスト (`cargo test`) と `cargo fmt` を必ず実行。
- フロントは Prettier で整形される標準スタイルを維持。
